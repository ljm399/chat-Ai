# 终端 `/` 指令选择与 `@` 文件上下文选择实现计划

## 1. 背景与目标

当前项目是一个基于 Node.js ESM 的终端 AI 助手，现有交互模式是：

- 使用 `readline.question()` 逐轮读取用户输入
- 把用户输入原样推入 `messages`
- 调用 OpenAI 接口获取模型回复

本次计划的目标是在不改变 OpenAI 请求职责边界的前提下，新增两类终端交互能力：

1. `/` 指令选择
   - 用户输入 `/` 时拉起指令列表
   - 选中某条指令后，把对应模板写回当前输入框
   - 支持单独输入 `/`，也支持在同一行里输入 `/`

2. `@` 文件上下文选择
   - 用户输入 `@` 时拉起项目文件列表
   - 用户选中一个文件后，将该文件内容作为“仅本轮上下文”
   - 文件上下文与本轮用户问题一起发给模型
   - 本轮完成后清空，不污染下一轮

## 4. 具体设计

### 4.1 输入模式重构

将 `src/app.js` 中的交互模式从一次性问答改为持续输入模式：

- 使用 `readline.emitKeypressEvents(process.stdin)`
- 若终端支持，则调用 `process.stdin.setRawMode(true)`
- 监听 `process.stdin.on("keypress", ...)`

需要维护统一输入状态对象，建议包含以下字段：

```js
{
  inputBuffer: "",
  cursorIndex: 0,
  picker: {
    visible: false,
    type: null,
    items: [],
    activeIndex: 0,
    triggerIndex: -1
  },
  selectedContextFile: null
}
```

字段说明：

- `inputBuffer`
  - 当前输入框中的完整文本
- `cursorIndex`
  - 当前光标所在字符位置
- `picker.visible`
  - 是否显示选择器
- `picker.type`
  - 当前选择器类型，值为 `slash` 或 `at`
- `picker.items`
  - 当前菜单项列表
- `picker.activeIndex`
  - 当前高亮项索引
- `picker.triggerIndex`
  - 触发 `/` 或 `@` 的字符索引，用于回填或移除触发字符
- `selectedContextFile`
  - 当前本轮附加上下文文件信息

### 4.2 终端渲染模块

新增：

- `src/utils/terminalRenderer.js`

职责：

- 清除上一次输入区渲染内容
- 渲染 prompt
- 渲染当前输入内容
- 根据 `cursorIndex` 恢复光标位置
- 渲染打开的选择器列表
- 渲染当前已选择的上下文文件提示

建议暴露接口：

```js
renderInputFrame(state)
clearInputFrame()
```

显示要求：

- prompt 统一显示为 `问：`
- 已选择上下文时显示：
  - `当前上下文: <relativePath>`
- 选择器打开时，显示最多若干条候选项
- 当前高亮项使用明显颜色区分

### 4.3 `/` 指令管理

新增：

- `src/utils/commandManager.js`

职责：

1. 提供内置固定指令
2. 扫描 `src/docs` 目录下的模板文件
3. 返回统一结构的命令列表

内置指令至少包括：

- `explain`
  - 内容示例：请解释下面的代码或问题，并给出关键点。
- `fix`
  - 内容示例：请帮我定位问题原因，并给出修复方案和修改建议。
- `refactor`
  - 内容示例：请对下面的代码进行重构，并说明重构收益。
- `summary`
  - 内容示例：请总结下面内容的重点，并给出简短结论。

返回结构：

```js
{
  id: "builtin:explain",
  label: "explain",
  description: "解释代码或问题",
  content: "请解释下面的代码或问题，并给出关键点。",
  source: "builtin"
}
```

动态模板规则：

- 扫描 `src/docs`
- 若目录不存在，直接返回空数组，不报错
- 仅读取文本类文件
- 模板名使用文件名
- 模板内容使用文件全文
- `source` 固定为 `docs`

对外接口：

```js
async function getSlashCommands()
```

### 4.4 `@` 文件扫描与上下文读取

新增：

- `src/utils/contextPicker.js`

职责：

1. 递归扫描当前项目文件
2. 过滤不应进入候选列表的目录和文件
3. 读取选中文件内容
4. 控制上下文大小

扫描规则：

- 根目录为 `process.cwd()`
- 排除目录：
  - `node_modules`
  - `.git`
  - `.front/history`
- 默认仅保留文本文件
- 排除明显二进制文件
- 排除超大文件

返回结构：

```js
{
  path: "C:\\Users\\MJL\\Desktop\\ai学习\\ai-project\\src\\app.js",
  relativePath: "src/app.js",
  size: 3113
}
```

建议对外接口：

```js
async function getProjectFiles()
async function readContextFile(filePath)
```

其中 `readContextFile(filePath)` 返回：

```js
{
  relativePath: "src/app.js",
  content: "...",
  truncated: false
}
```

上下文大小控制：

- 读取文件时设置字符上限
- 超限则截断
- 返回 `truncated: true`
- 在界面上给出提示，例如：
  - `文件过大，已截断后发送`

### 4.5 `/` 触发规则

触发条件按当前输入与光标位置判断：

- 单独输入 `/` 时直接拉起菜单
- 行内输入 `/` 时也允许触发

本次实现采用更实用的解释方式：

- `/` 位于行首或任意单词位置时都可触发
- 若用户正在输入 `/`，立即打开指令菜单
- 选中后用命令内容替换当前触发位置上的 `/`

这样做的理由：

- 更符合实时终端交互
- 避免严格依赖“前后没有空格”的复杂字符判断
- 与本次“方向键即时菜单”目标一致

如果后续必须严格限制字符邻接规则，再补充更细的判断逻辑。

### 4.6 `@` 触发规则

触发方式：

- 单独输入 `@` 时拉起文件列表
- 行内输入 `@` 时也拉起文件列表

选中后行为：

- 单独触发：
  - 保留空输入框
  - 仅记录当前本轮上下文文件
- 行内触发：
  - 删除输入中的 `@`
  - 保留其它文字
  - 记录当前本轮上下文文件

注意：

- 当前版本只支持单文件上下文
- 再次触发 `@` 选择新文件时，覆盖旧的 `selectedContextFile`

### 4.7 用户消息拼装

新增一个消息构建函数，建议放在 `src/app.js` 内，或者单独放入工具模块。

建议接口：

```js
async function buildUserMessage({ rawInput, selectedContextFile })
```

行为：

- 若没有上下文文件，直接返回 `rawInput`
- 若有上下文文件，先读取文件内容，再按固定模板拼接：

```text
以下是本轮附加的项目文件上下文：
文件路径: <relativePath>
文件内容：
<content>

用户问题：
<rawInput>
```

消息历史规则：

- `messages` 中写入的是最终拼装后的 user message
- 不额外插入一条独立的 context message
- AI 回复照常入栈
- 本轮请求结束后清空 `selectedContextFile`

## 6. 测试与验收计划

实现后至少验证以下场景：

1. 普通提问
   - 不使用 `/` 或 `@`
   - 行为与当前一致

2. 单独输入 `/`
   - 成功拉起指令菜单
   - 可用方向键切换
   - 回车后模板进入输入框

3. 行内输入 `/`
   - 在同一行触发菜单
   - 选中后用模板替换 `/`
   - 可继续编辑并发送

4. 单独输入 `@`
   - 成功拉起文件列表
   - 选中文件后仅记录上下文，不立即发送
   - 继续输入问题后可正常发送

5. 行内输入 `@`
   - 拉起文件列表
   - 选择文件后移除 `@`
   - 其余输入内容保留

6. 上下文仅本轮有效
   - 当前轮请求包含文件上下文
   - 下一轮未重新选择文件时，不再自动附加旧文件内容

7. 大文件截断
   - 超限文件会被截断
   - 有明确提示

8. 文件过滤
   - `node_modules`
   - `.git`
   - `.front/history`
   - 以上内容不出现在候选列表中

9. 退出逻辑
   - `exit`
   - `quit`
   - `Ctrl+C`
   - 均能正常退出并保存历史

10. `src/docs` 动态命令
   - 目录不存在时 `/` 依旧可用
   - 放入模板文件后，菜单中可出现新增命令

## 7. 风险与注意事项

### 7.1 终端兼容性

风险：

- Windows PowerShell 的原始模式、光标恢复、ANSI 清屏行为可能与其它终端略有差异

应对：

- 渲染逻辑尽量简单
- 不依赖复杂全屏终端库
- 采用“清当前输入区 + 重绘”的保守方案

### 7.2 文本文件识别

风险：

- 简单后缀判断可能把部分非文本文件误判为文本

应对：

- 先基于扩展名和内容采样做基础过滤
- 当前版本优先保证可用性，后续再增强

### 7.3 上下文体积

风险：

- 文件内容过大时会影响模型输入长度和响应速度

应对：

- 增加截断上限
- 将截断提示展示给用户
